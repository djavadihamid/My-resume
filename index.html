What I Know & what I Can do

Unity ECS + Burst Compiler + Job System (DOTS) - Familarity

Recently Unity introduced these stuff. If you are familiar with them, you will absolutely know that the future of game development in Unity would be done by them!

Unity C# Development - Game Logic

You know! Writting code is not that hard! Everyone can do that; But writting reusable structural components and plugins using design patterns like observer, facade, template, command and etc need experience.

Unity 2D & 3D development

Another important basis of developing a game in unity is to have the ability to work with Unity editor. making a responsive UI, organized hierarchy, scene composition, lighting, animation system and so on are all vital.

Online game development (real-time and non-real-time)

3 years ago, I was working as a backend developer with PHP and Laravel. This experience helps me to understand and help backend developers to make APIs and even making a real-time game using socket.

Extending editor

A handy tool can definitely accelerate the development process. If i am not exaggerating, It it as important as business implementation. By editor scripting, Unity can handle lots of jobs that maybe developers do without automation.

Handling data via local Database (Sqlite)

Handling and saving all data with playerPrefs is not that wise; Also, working with a database is not as easy as playerPrefs. However it can be a peace of cake with 3-layer structure (DataAccess-Entity-Service).

The art of logging

"Where is the bugs?! I cannot find them! I have to spend time to find a simple bug!!!"
Familiar dialogs uh? But knowing how to print pretty logs can help teams to save time finding and resolving bugs and errors as fast as possible.

External SDKs

Nowadays game stores and Ads agencies have great roles in publishing a game. Some has neat SDKs and some not. But experience working with them would be a great help.

Git VCS

Without Git? Imposibble to do a single action! working with Git in a team is essential. Resolving conflicts, branching, merging and so on are serious stuffs to work with in a team.

Documentation

An understandable document is essential for working in team and writting an extendable, maintainable component. Doing so in a neat stylized Html page can make life easier for developers to use that. There is also some sort of documents like gdd, proposal, pitch and etc that are important to a project.

Game Design

Nowadays, preproduction is an undeniable part of game development; And the major part of that, is game design. It is a set of rules that the game will be created based on. Risk reward, Bartle taxonomy, conditional rules and etc are a few rules that should be implemented in game design.
